# 近5天增长监控报告（基于 grow_base 阈值）

## 目标

对指定项目做“指定日期范围（默认近5个完整自然日）”监控，按国家输出：
- 阈值对比（ROI0/CPI/次留）
- 风险分层（高风险/中风险/健康）
- eCPM 分广告位观察（插屏/AppOpen/激励）
- 可执行动作建议（P0/P1/P2）

并落地两份文件：
- `monitor_report/grow/{项目名}_{MMDD}.md`
- `monitor_report/grow/{项目名}_{MMDD}.html`

## 输入要求

- `project_id`（如 `A124`）与项目名（如 `MessagePro`）。
- 基准文件：`grow_base/{project_id}_{项目名}.md`。
- 国家范围：默认 `US/AU/CA/GB/FR/DE`；若基准文件有明确国家列表，以基准为准。
- 日期范围（可选）：`date_start`、`date_end`。
  - 默认：近 5 个完整自然日（`今天-5` 到 `今天-1`）。
  - 若用户明确指定日期范围，按用户范围执行。

## 时间口径

- 默认使用“近 5 个完整自然日”：`今天-5` 到 `今天-1`。
- 若传入 `date_start/date_end`，以用户日期为准。
- 不使用当天数据，避免实时回传不全导致偏差。
- 报告中必须写明绝对日期范围。

## 数据抓取（snowball_mcp）

### 1) 项目与阈值确认
- 从 `grow_base/{project_id}_{项目名}.md` 读取国家阈值：`ROI0`、`CPI`、`次留`。
- 阈值仅使用 grow_base 已发布值，不临时重算。

### 2) 监控期国家投放与回收数据
- 对每个国家调用 `get_roi_report`：
  - `day_period=daily`
  - `dimension_list=["date","country"]`
  - `roi_fields=["roi_0"]`
  - `date_start=监控起始日`
  - `date_end=监控结束日`
- 聚合字段：`costs`、`cohort_revenue`、`roi0.rate`、`cpi`、`retention_1`。

### 3) eCPM 分广告位
- 调用 `get_ads_revenue_table_data`，拉取目标国家、目标日期范围。
- 建议口径：按日期范围聚合到国家+广告位，计算加权 eCPM：
  - `eCPM = Σads_revenue / Σimpressions * 1000`
- 重点输出：`Interstitial`、`AppOpen`、`RewardedVideo`（若样本不足标记“样本不足”）。

## 计算与判定规则

### 1) 核心聚合
- `监控期ROI0`：监控期加权或聚合均值（口径在报告中写清）。
- `监控期CPI`：监控期聚合值（`Σcost / Σinstall`）或接口聚合输出。
- `次留均值`：仅用有值天数计算，缺失日不计入分母。
- `监控期花费`：`Σcosts`。
- `监控期营收`：`Σcohort_revenue`。
- 展示格式：所有 ROI 指标（如 `监控期ROI0`、`阈值ROI0`、`ROAS`）统一转为百分比 `x%` 展示。

### 2) 状态分层（按国家）
- `高风险`：三项中任意两项不达标（ROI0 低于阈值 / CPI 高于阈值 / 次留低于阈值）。
- `中风险`：仅一项不达标。
- `健康`：三项全部达标。

### 3) 总体结论
- 汇总 6 国花费、营收、cohort ROAS。
- 输出健康分层清单与“是否可扩量”结论。

## 输出格式

### A. Markdown 报告
结构固定：
1. 标题：`# {项目} 近5天增长运营监控（买量+营收）`
2. 监控范围（项目、日期、阈值来源）
3. 总体健康结论
4. 国家阈值和核心指标监控表（国家一行）
5. eCPM 分广告位监控表
6. 异常与风险定位
7. 运营动作建议（P0/P1/P2）
8. 结论摘要

### B. HTML 报告
- 采用 grow_base 系列报告的卡片 + 表格风格。
- 内容与 MD 一一对应，不新增口径。
- 在“总体健康结论”下必须展示核心检查指标，且第 2 项固定为 `日均花费`。
  建议顺序：`总花费`、`日均花费`、`cohort营收`、`cohort ROAS`。
- 国家表中核心数值着色规则与状态色一致：
  - 风险值用 `高风险` 同款红色（如 ROI0 低于阈值、CPI 高于阈值、次留低于阈值）。
  - 健康值用 `健康` 同款绿色（如 ROI0 达标、CPI 达标、次留达标）。
- 页脚写明 `Generated from {md绝对路径}`。

## 文件命名规范（强制）

- 必须使用：`{项目名}_{MMDD}`，例如：
  - `A124_MessagePro_0312.md`
  - `A124_MessagePro_0312.html`
- 保存目录：`monitor_report/grow/`。

## 质量检查清单

发布前必须检查：
- 日期是否符合输入口径（未指定时为“今天-5 到 今天-1”）。
- MD 与 HTML 的核心数值是否一致（花费、营收、ROAS、国家表）。
- ROI 相关字段是否全部使用百分比展示（禁止小数原值如 `0.4172`）。
- 阈值是否来自 `grow_base` 文件而非临时估值。
- HTML `Generated from` 是否指向当次 `_MMDD.md` 文件。
- eCPM 样本不足是否已明确标注。

## 约束

- 只基于 snowball_mcp 返回数据与 grow_base 已有阈值；禁止补数或猜测。
- 禁止输出“全局均值替代国家结论”。
- 若某国家关键字段缺失，必须在报告中显式标注“数据缺失/低置信度”。
