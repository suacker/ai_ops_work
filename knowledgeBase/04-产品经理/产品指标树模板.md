# 产品指标树模板

> 适用于建立产品经营看板、定义版本目标、拆解留存/变现/买量回收问题。指标树必须能从北极星指标下钻到国家、平台、版本、渠道、人群和功能模块。

| 负责人 |  |

## 北极星指标

| 指标 | 定义 | 为什么代表用户价值 | 为什么代表商业价值 | 不适用场景 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

选择原则：

- IAA 产品优先选择能同时代表使用深度和广告库存质量的指标，例如有效会话、核心行为完成用户、广告可承接 DAU。
- 订阅产品不能只选收入，必须连接 trial、付费后使用、续订和退款。
- 混合变现产品要避免单一收入指标掩盖免费用户体验恶化。

## 一级指标树

| 目标 | 一级指标 | 公式/口径 | 主要 Owner |
| --- | --- | --- | --- |
| 用户规模 | DAU / WAU / MAU / New Users |  | PM + 增长 |
| 激活 | Onboarding 完成率 / 首次核心行为率 |  | PM |
| 留存 | D1 / D3 / D7 / D30 |  | PM + 数据 |
| 使用深度 | 核心行为次数 / 会话时长 / 功能渗透率 |  | PM |
| 订阅/IAP | Paywall CVR / Trial Start / Purchase / Renewal / Refund |  | PM + 商业化 |
| IAA | Ads ARPDAU / IPU / eCPM / Impressions per User |  | PM + 变现 |
| 买量回收 | CPI / ROI0 / ROI7 / LTV / cohort profit |  | 增长 + PM |
| 质量 | Crash Free / ANR / 支付失败 / 广告失败 / 客诉 |  | 技术 + QA |

## 二级拆解

### 激活

| 指标 | 定义 | 常见问题 | 可行动作 |
| --- | --- | --- | --- |
| 首次打开率 | 安装后成功打开产品的比例 | 包体、归因、启动崩溃 | 技术排查、渠道对账 |
| Onboarding 完成率 | 开始 onboarding 后完成关键步骤的比例 | 步骤过长、权限过早、价值表达弱 | 简化步骤、调整权限时机、文案实验 |
| 首次核心行为率 | 新用户完成第一个核心任务的比例 | 价值不清、入口隐藏、任务失败 | 新手引导、默认模板、错误兜底 |

### 留存

| 指标 | 定义 | 拆解维度 | PM 判断 |
| --- | --- | --- | --- |
| D1 | 安装次日回访 | 国家、渠道、版本、onboarding 分组 | 首日价值是否清晰 |
| D7 | 安装第 7 日回访 | 国家、渠道、核心行为次数 | 是否形成可重复场景 |
| D30 | 安装第 30 日回访 | 付费状态、功能渗透、push 触达 | 是否有长期价值 |

### 订阅/IAP

| 指标 | 定义 | 拆解维度 | PM 判断 |
| --- | --- | --- | --- |
| Paywall 展示率 | 进入付费墙用户 / 目标用户 | 入口、国家、版本、人群 | 展示时机是否足够且不过早 |
| Paywall 点击率 | 点击价格包或 CTA / 展示 | 价格包、文案、权益 | 付费承诺是否清晰 |
| Trial Start | 开始试用 / 展示或点击 | 国家、价格、入口 | 试用吸引力是否足够 |
| Trial to Paid | 试用转付费 / 试用开始 | 周期、权益使用、取消原因 | 试用期是否兑现价值 |
| 续订率 | 成功续订 / 到期订阅 | 周期、国家、用户层 | 长期价值是否成立 |
| 退款率 | 退款 / 购买 | 国家、入口、客服原因 | 承诺是否过度或体验不达预期 |

### IAA

| 指标 | 定义 | 拆解维度 | PM 判断 |
| --- | --- | --- | --- |
| Ads ARPDAU | 广告收入 / DAU | 国家、广告格式、场景、版本 | 广告收入效率 |
| IPU | 展示次数 / DAU 或广告用户 | 广告格式、场景、人群 | 广告库存深度 |
| eCPM | 广告收入 / 展示 * 1000 | 国家、广告源、格式 | 流量质量和广告填充价值 |
| 广告后退出率 | 广告关闭后退出会话 / 广告展示 | 场景、频次、用户层 | 广告是否破坏体验 |

### 买量回收

| 指标 | 定义 | 拆解维度 | PM 判断 |
| --- | --- | --- | --- |
| CPI | Spend / Installs | 国家、network、campaign、adgroup | 买量成本是否可承接 |
| ROI0 / ROI7 | cohort revenue / spend | 国家、渠道、campaign | 新用户质量和回收速度 |
| cohort profit | cohort revenue - spend | 国家、渠道、版本 | 是否具备持续放量条件 |

## 维度标准

| 维度 | 必填原因 | 示例字段 |
| --- | --- | --- |
| 日期 | 判断趋势和实验窗口 | date |
| 国家 | 出海产品 ROI、eCPM、支付差异大 | country |
| 平台 | iOS/Android 变现和质量不同 | platform |
| 版本 | 判断版本影响 | app_version |
| 渠道/network | 买量质量和归因差异 | network |
| campaign/adgroup | 连接投放动作 | campaign, adgroup |
| 用户状态 | 区分免费、试用、付费、取消 | user_status |
| 实验分组 | 判断因果 | experiment_id, variant |

## 指标诊断顺序

1. 先确认数据质量：日期、版本、国家、渠道、采样、null、-1 总计行。
2. 再判断是否是规模变化：DAU、新用户、买量 spend、渠道结构。
3. 再拆留存和核心行为：激活、D1、D7、核心行为频次。
4. 再拆商业化：订阅/IAP 或广告格式、场景、人群。
5. 最后连接买量回收：CPI、ROI、cohort profit 是否支持动作。
