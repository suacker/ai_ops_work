## 埋点目标

| 目标 | 需要回答的问题 | 关联指标 |
| --- | --- | --- |
| 激活 | 新用户是否完成首次价值体验？ | onboarding 完成、首次核心行为 |
| 留存 | 哪些行为驱动 D1/D7？ | 核心行为频次、功能渗透 |
| 变现 | 用户是否理解并接受付费/广告？ | Paywall CVR、Trial、Ads ARPDAU |
| 买量回收 | 哪些渠道用户质量更好？ | ROI、LTV、cohort profit |
| 质量 | 是否存在版本或链路异常？ | crash、失败率、投诉 |

## 事件命名规范

| 类型 | 规范 | 示例占位 |
| --- | --- | --- |
| 页面展示 | `{module}_page_show` | `onboarding_page_show` |
| 按钮点击 | `{module}_{action}_click` | `paywall_purchase_click` |
| 流程结果 | `{module}_{action}_result` | `purchase_result` |
| 广告事件 | `ad_{format}_{action}` | `ad_interstitial_show` |
| 订阅事件 | `sub_{action}` | `sub_trial_start` |
| 实验曝光 | `experiment_exposure` |  |

命名要求：

- 使用小写字母、数字和下划线。
- 同一含义只允许一个事件名。
- 枚举值必须提前列全，不允许上线后自由写字符串。
- 客户端、服务端和数据看板口径必须能对齐。

## 事件验收

| 检查项 | 标准 | 结果 |
| --- | --- | --- |
| 触发时机 | 与 PRD 定义一致，不重复不漏发 |  |
| 参数完整 | 必填参数不为空，枚举值合法 |  |
| 分组正确 | 实验用户只进入一个 variant |  |
| 漏斗可跑 | 关键路径每一步都有事件 |  |
| 收入口径 | 订阅/IAP/广告能与平台或 Snowball 对齐 |  |
| 版本下钻 | 能按 app_version 排查异常 |  |

## 数据质量风险

| 风险 | 表现 | 处理 |
| --- | --- | --- |
| 重复上报 | 指标异常偏高 | 增加幂等或去重字段 |
| 漏报 | 漏斗断层 | 对照客户端日志和服务端回调 |
| 参数漂移 | 枚举值不一致 | 上线前冻结字典 |
| 当前日采样 | 当天数据波动 | 完整自然日报告排除或标注低置信度 |
| 归因缺失 | 无法连接买量 | 检查 network/campaign/adgroup |

## 上线后监控

| 时间点 | 检查内容 | 处理 |
| --- | --- | --- |
| T+0 | 事件是否有量、参数是否完整 | 异常则阻断放量 |
| T+1 | 漏斗趋势、版本质量、收入事件 | 异常则回滚或修复 |
| T+3 | 实验分组、留存、变现初步结果 | 决定扩量或继续观察 |
