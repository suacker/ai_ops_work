# 常见分析 SOP

## 通用分析流程

| 步骤 | 动作 | 输出 |
| --- | --- | --- |
| 1 | 确认产品、Project ID、平台、国家、渠道和绝对日期范围 | 分析范围 |
| 2 | 确认主数据源和辅助数据源 | 数据源列表 |
| 3 | 检查数据质量：缺失、采样、预测、`-1` 总计行、null、时区 | 可信度等级 |
| 4 | 先看公司/产品层，再下钻国家、渠道、Campaign、adgroup、广告格式或版本 | 分层结论 |
| 5 | 区分事实、推断和建议 | 结论可信度 |
| 6 | 输出 P0/P1/P2 动作、Owner、复查指标和复查日期 | 动作闭环 |
| 7 | 完成后沉淀复盘或案例 | 可复用经验 |

## 买量诊断 SOP

1. 确认产品、Project ID、渠道、账户和绝对日期范围。
2. 先做平台与 Snowball/BI 对账：spend、install、campaign 命名、国家、时区。
3. 看账户和国家层 spend、install、CPI、ROI、留存。
4. 下钻 campaign/adgroup/adset，区分成熟放量、测试预算、衰退和数据缺失。
5. 对素材只做基于 parent campaign/adgroup 回收质量的推断，不能写直接 creative ROI。
6. 按 `ready`、`partial`、`blocked` 判断是否允许给预算动作。
7. 输出 P0/P1/P2 动作和复查日期。

## 广告变现异常 SOP

1. 确认异常日期和对比基线，默认不包含当天。
2. 拆 DAU/DNU、impressions、ads revenue、ARPDAU、eCPM、IPU。
3. 拆广告格式：AppOpen、Interstitial、RewardedVideo、Banner、Native。
4. 拆广告场景和 mediation source：AdMob、AppLovin、其他 network。
5. 检查流量结构、展示机会、填充、eCPM、广告格式结构、配置和版本变化。
6. 区分 gross loss、补偿收入、net loss。
7. 输出修复动作、观察窗口和防复发监控。

## iOS 订阅诊断 SOP

1. 单独抽 iOS，不和 Android IAA 混合下结论。
2. 拆 Paywall 曝光、点击、试用、首购、续订、取消、退款。
3. 按国家、版本、入口、价格、用户来源和订阅周期下钻。
4. 结合买量 cohort 判断订阅 LTV 和回收。
5. 等待完整试用周期后再判断 Trial Conversion。
6. 输出 Paywall、价格、权益、流量质量或退款治理动作。

## 产品实验 SOP

1. 写清实验假设，明确“如果成功，将做什么动作”。
2. 定义主指标、辅助指标和护栏指标。
3. 写清分组、样本、周期、上线条件和停止条件。
4. 实验中只看监控和护栏，避免提前宣布胜负。
5. 结束后必须复盘并沉淀到案例库。

## 输出检查

- [ ] 是否写了绝对日期和时区。
- [ ] 是否写清主数据源和辅助数据源。
- [ ] 是否处理了预测值、采样当前日、`-1`、null、N/A。
- [ ] 是否拆到足够支持动作的维度。
- [ ] 是否明确不能判断的范围。
- [ ] 是否给出 Owner、截止时间、复查指标。
