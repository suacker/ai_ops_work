# 数据源地图

## 数据源优先级

| 场景 | 主数据源 | 辅助数据源 | 决策边界 |
| --- | --- | --- | --- |
| 买量回收 | Snowball / BI | Meta Ads / Google Ads | 预算动作以回收质量为准 |
| 平台投放状态 | Meta Ads / Google Ads | Snowball / BI | 看 campaign/adset/ad 状态和素材 |
| 广告变现 | Snowball / AdMob / AppLovin | BI 报告 | 收入异常需拆广告格式和 mediation |
| iOS 订阅 | App Store Connect / BI | Snowball | 关注试用、续订、退款和订阅 cohort |
| Android IAP | Google Play Console / BI | Snowball | 关注购买、退款和国家 |
| 用户行为漏斗 | Snowball / BI | Firebase / 埋点后台 | 需排除采样当天 |
| 产品质量 | Crash / ANR / 用户反馈后台 | BI / 工单 | 影响留存和收入时进入经营看板 |

## 对账顺序

| 冲突 | 优先检查 | 常见原因 | 临时处理 |
| --- | --- | --- | --- |
| 平台花费和 BI 花费不一致 | 时区、账户、campaign 命名、汇率、退款/抵扣 | 平台和 BI 结算口径不同 | 标注差异，不做强预算动作 |
| 平台 install 和 BI install 不一致 | 归因窗口、SKAN、渠道映射、事件回传 | 回传延迟或归因断裂 | 先做归因质量检查 |
| AdMob 与 AppLovin 收入不一致 | 主口径、时区、预估/最终收入、mediation source | 后台预估和最终收入延迟 | 标注预估，次日复查 |
| App Store Connect 与 BI 订阅不一致 | 订单时间、退款、试用转正、订阅周期 | 订阅事件延迟或退款处理不同 | 拆订单和用户口径 |
| 漏斗事件异常 | 版本、事件名、采样、埋点条件 | 埋点丢失或事件定义变化 | 暂停实验结论 |

## 报告引用规则

- 主结论必须引用主数据源。
- 辅助数据源只用于解释，不直接替代主数据源。
- 数据源冲突时必须写“冲突项、差异大小、暂定口径、复查时间”。
- 无法对账时，动作状态应降级为 `blocked` 或 `partial`。
