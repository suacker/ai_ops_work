# 分析报告交付标准

## 适用范围

适用于买量诊断、变现异常、订阅漏斗、产品实验、运营活动、重大复盘和经营周报。

## 必填结构

| 模块 | 要求 |
| --- | --- |
| 标题 | 产品、主题、绝对日期范围 |
| 基本信息 | Project ID、平台、国家、负责人、生成日期 |
| 数据源与口径 | 主数据源、辅助数据源、查询时间、时区、是否 forecast |
| 5 行结论 | 每行都必须有经营含义 |
| 核心 KPI | 当前值、对比值、变化、阈值、结论 |
| 维度下钻 | 至少拆到能支持动作的层级 |
| 异常定位 | 事实、推断、不能判断范围分开写 |
| 动作建议 | P0/P1/P2、Owner、截止日期、复查指标 |
| 方法与置信度 | 样本量、缺失、预测、采样、数据质量等级 |
| 附录 | 查询参数、截图、链接、原始表路径 |

## Markdown 与 HTML

| 交付物 | 要求 |
| --- | --- |
| Markdown | 作为真值来源，所有数值和结论先写入 Markdown |
| HTML | 从同一组中间数据生成，与 Markdown 数值一致 |
| 页脚 | HTML 必须包含 `Generated from {Markdown 绝对路径}` |
| 表格 | 可横向滚动或响应式展示，不得截断关键列 |

## 结论写法

| 类型 | 推荐写法 | 避免写法 |
| --- | --- | --- |
| 事实 | `2026-06-20 ~ 2026-06-26 A155 US facebook ROI0 为 xx%` | `最近表现还行` |
| 推断 | `基于所属 campaign/adgroup 回收质量推断该素材可继续观察` | `该 creative ROI 很高` |
| 风险 | `数据源存在 install 差异，预算动作降级为 partial` | `数据可能有点问题` |
| 动作 | `P1: Owner 在 YYYY-MM-DD 前补跑国家维度并复查 ROI0` | `建议关注` |

## 最终检查

- [ ] 绝对日期、时区、Project ID 齐全。
- [ ] 指标格式符合 [[指标字典]]。
- [ ] 没有把预测值、null、N/A、`-1` 当作真实 0。
- [ ] 动作建议符合 [[动作分级与决策边界]]。
- [ ] 数据质量问题已登记到 [[数据质量治理机制]]。
- [ ] 可复用经验已进入 `08-复盘与案例库/`。
