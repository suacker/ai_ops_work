# 复盘会议 SOP

## 何时必须复盘

| 场景 | 复盘时限 |
| --- | --- |
| P0 异常影响收入、花费、利润、版本或数据可信度 | 异常关闭后 3 个工作日内 |
| 高花费 Campaign 放量成功或失败 | 完整观察窗口后 3 个工作日内 |
| 重要素材成为 winner 或失效 | 判断成立后 1 周内 |
| 广告变现配置造成显著收入变化 | 配置稳定后 3 个完整自然日内 |
| Paywall、价格、订阅权益实验结束 | 实验结束并完成数据读取后 |
| 同类问题重复出现 2 次以上 | 下次周会前 |

## 会前准备

| 角色 | 准备内容 |
| --- | --- |
| Owner | 背景、时间线、动作记录、结果数据 |
| BI | 主数据源、对比周期、口径说明、数据质量等级 |
| 协作方 | 执行记录、配置变更、素材/版本/权限截图 |
| 主持人 | 复盘目标、会议议程、决策边界 |

## 会议流程

| 环节 | 问题 | 输出 |
| --- | --- | --- |
| 事实对齐 | 发生了什么，影响多少 | 时间线和影响评估 |
| 判断还原 | 当时为什么这样判断 | 决策依据和约束 |
| 根因分析 | 为什么有效或失败 | 直接原因和深层原因 |
| 复用边界 | 哪些地方能复制，哪些不能 | 适用条件和禁止套用条件 |
| 机制更新 | 哪些 SOP、指标、权限、监控要改 | 具体动作和 Owner |

## 会后产出
