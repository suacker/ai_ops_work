# 02 买量增长

## 定位

本空间沉淀出海 App 买量增长方法论，覆盖 Google Ads、Facebook / Meta Ads 的账户结构、投放实验、国家放量、素材迭代、预算迁移、归因对账和买量用户质量判断。

这里不是实时数据报告区，不记录未经查询确认的实时结论。所有页面默认只写方法、模板、字段、决策规则和示例占位。实际预算和素材动作必须结合 Snowball / BI 回收，不得只看 CTR、CPI、平台 ROAS 或单日波动。

## 使用顺序

1. 确认产品、Project ID、商业模式和目标国家。
2. 先做归因和数据质量检查，判断 `ready / partial / blocked / inactive_no_data`。
3. 用 [[广告系列生命周期诊断模板]] 确认通用阶段和动作框架，量化阈值和单主动作规则看 [[Google_Meta_Campaign生命周期投放优化决策指南]]，再进入 Google 或 Meta 平台手册。
4. 日常例行判断直接走 [[买量每日决策SOP与参数表]] 的六步流程和日报规范。
5. 用国家、campaign、adgroup/adset、素材、用户质量模板完成下钻。
6. 最后写入预算迁移建议和复查计划，默认不直接执行平台写操作。

## 核心对象

| 层级 | Google Ads | Meta Ads | Snowball / BI 对应口径 |
| --- | --- | --- | --- |
| Account | customer_id | ad account | account / network |
| Campaign | App campaign / PMax | ASC / manual campaign | campaign |
| 子层级 | asset group / ad group | adset | adgroup |
| 素材 | asset / video / image / text | ad / creative | 通常只能映射到 parent campaign/adgroup |
| 用户 | install cohort / conversion | install cohort / event | install cohort、留存、事件、收入 |

## 决策门禁

| 门禁 | 通过标准 | 不通过时的处理 |
| --- | --- | --- |
| 数据质量 | Spend、install、campaign、country 能对齐，核心回收字段可用 | 只给 `data_gap_check`，不做预算强建议 |
| 样本量 | Spend、install、事件样本达到当前产品阈值或测试约定 | 标记 `低置信度` 或 `too early to judge` |
| 回收质量 | ROI0 / ROI3 / ROI7、利润、留存至少一个完整观察窗达标 | 不因 CTR 高或 CPI 低直接放量 |
| 变现匹配 | IAA 看广告收入、eCPM、IPU、留存；IAP 看付费率、ARPPU、订阅/首购 | 缺字段时明确写 `数据缺失` |
| 操作授权 | 用户明确授权并二次确认具体对象 | 默认只输出建议和复查日期 |

## 动作词表

买量 9 个标准动作（`add_budget / small_add / maintain / test_budget / reduce_or_reallocate / pause_candidate / refresh_creative / duplicate_restart / data_gap_check`）的含义、进入条件和默认复查窗口，以 [[动作分级与决策边界]] 为唯一定义，本空间所有页面只引用、不重复定义。保护窗（学习、重大编辑、操作恢复、审核恢复、转化滞后、异常事件）和单主动作原则同样以该页为准。Google 平台子动作（`add_assets`、`refresh_video`、`split_asset_group`，均属 `refresh_creative` 细分）见 [[Google投放与诊断手册]]。
