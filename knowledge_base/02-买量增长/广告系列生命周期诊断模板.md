# 广告系列生命周期诊断模板

## 1. 文档定位

本页是 Google Ads 与 Meta / Facebook Ads 的广告系列生命周期诊断总模板，用于统一“一个 Campaign 当前处于什么阶段、该观察还是放量、该刷新素材还是迁移预算、是否进入暂停候选”的判断体系。

平台差异不在本页展开，必须跳转到对应平台手册：

| 平台 | 平台手册 | 使用场景 |
| --- | --- | --- |
| Google Ads | [[Google投放与诊断手册]] | App Campaign、App Engagement、效果最大化、tCPA、tROAS、素材资产、资产组、国家扩量 |
| Meta / Facebook Ads | [[Meta投放与诊断手册]] | Advantage+ App Campaign、手动 Campaign、CBO/ABO、adset、creative、Learning、频次和素材疲劳 |

动作词表和数据质量状态的唯一定义在 [[动作分级与决策边界]]，本页与平台手册只引用、不另行定义。

本页只解决 4 个通用问题：

1. 这个广告系列当前处于哪个生命周期阶段。
2. 当前结论是否有足够实时数据支持。
3. 平台信号和 Snowball / BI 回收是否一致。
4. 下一步动作、负责人和复查窗口是什么。

## 2. 实时性要求

生命周期诊断必须基于当前窗口和当前状态。历史经验只能作为假设来源，不能替代实时查询。

| 必须刷新 | 说明 | 不能接受 |
| --- | --- | --- |
| 当前日期窗口 | 默认使用最近 7 个完整自然日，不含当天；特殊场景写明绝对日期 | 只写“最近几天” |
| 当前广告系列状态 | active、learning、limited、paused、rejected、budget limited 等 | 用上周状态推断当前仍有效 |
| 当前预算和出价 | 日预算、总预算、最近一次预算/出价/目标事件变更 | 用旧预算动作解释当前波动 |
| 当前素材表现 | 素材、资产、creative、频次、CTR、CPC、CPM、CVR、审核状态 | 直接沿用历史 winner |
| 当前国家与版位结构 | 国家、平台、placement、device、asset group/adset 分配 | 只看账户总均值 |
| 当前 Snowball / BI 回收 | spend、install、ROI0/3/7、profit、留存、广告/IAP 收入拆分 | 只看平台 ROAS 或平台 CPI |
| 当前归因映射 | campaign/adgroup/adset 命名、network、country、project_id | 新建或改名后继续套旧映射 |

若任一关键项缺失，先把数据质量降级为 `partial` 或 `blocked`，不得给强放量或强暂停建议。

## 3. 使用顺序

1. 确认产品、Project ID、商业模式、平台、账户和绝对日期范围。
2. 进入平台手册（[[Google投放与诊断手册]] / [[Meta投放与诊断手册]]），拉取当前平台字段。
3. 对齐 Snowball / BI：花费、安装、国家、campaign、adgroup/adset、收入和 ROI。
4. 判定数据质量：`ready / partial / blocked / inactive_no_data`。
5. 用本页生命周期矩阵给出阶段：`testing / scaling / mature / fatigue / pause_candidate`。
6. 给出动作：`test_budget / maintain / small_add / add_budget / refresh_creative / reduce_or_reallocate / pause_candidate / data_gap_check`。
7. 写入负责人、复查日期、复查指标和关闭标准。

## 6. 生命周期定义

| 状态 | 通用判断标准 | Google 特别注意 | Meta 特别注意 | 默认动作 |
| --- | --- | --- | --- | --- |
| testing | 新 campaign、新国家、新素材、新目标事件，样本尚未完成观察窗 | App Campaign / 效果最大化需要考虑转化延迟、出价策略和素材资产供给 | Learning 中或新 adset / creative 样本不足，不提前判 winner | `test_budget` / `maintain` |
| scaling | 回收达标，增量预算没有明显拉低国家、素材或用户质量 | tCPA / tROAS 目标仍合理，国家结构和资产组没有向低质流量漂移 | 频次未异常，Learning 稳定，素材池仍有新鲜度 | `small_add` / `add_budget` |
| mature | 多个完整窗口稳定，成本、回收、留存、素材表现波动可解释 | 需要持续补素材资产，避免资产组主题老化 | 需要持续补 creative 变体，避免频次和 fatigue 累积 | `maintain` / `refresh_creative` |
| fatigue | 成本上升、CTR/CVR 下降、ROI/profit/留存恶化或素材承诺错配 | asset label 变弱、素材覆盖不足、国家 mix 变差 | frequency 上升、CTR/CVR 下降、creative fatigue、版位/国家质量变差 | `refresh_creative` / `reduce_or_reallocate` |
| pause_candidate | 连续多个完整窗口不达标，且排除数据质量、归因延迟、学习期和样本不足 | 不因短期 tROAS 波动直接停；先确认 conversion value 和 Snowball revenue | 不因单日 CPI/CTR 或 Learning Limited 直接停；先确认回收和素材修复空间 | 人工确认后 `pause_candidate` |

补充说明：

- `learning` 不是第 6 个生命周期阶段，而是平台学习期的附加标记，输出可写成 `testing + learning`、`scaling + learning_risk` 等组合（Google 触发项见 [[Google投放与诊断手册]]，Meta 的 Learning / Learning Limited 见 [[Meta投放与诊断手册]]）。
- `inactive_no_data` 不是生命周期阶段，而是数据质量状态：对象无有效投放或无可用数据时输出 `no_action_no_data`，不做阶段判断。

## 7. 数据质量门禁

| 数据质量 | 判断条件 | 允许动作 | 禁止动作 |
| --- | --- | --- | --- |
| ready | 平台 spend/install/campaign/country 与 Snowball / BI 能对齐，核心回收字段可用 | P0/P1/P2 动作均可给，但平台写操作仍需授权 | 自动执行预算、暂停、出价或创建广告 |
| partial | 关键指标存在，但存在归因、映射、样本、forecast 或收入延迟问题 | `maintain`、`test_budget`、`refresh_creative`、中低置信度 `small_add` | 强放量、强暂停 |
| blocked | 花费、安装、campaign、国家或收入口径不可信 | `data_gap_check` | 预算、暂停、放量、降量结论 |
| inactive_no_data | 对象无有效消耗或无可用数据 | `no_action_no_data` | 做投放好坏判断 |

## 8. 通用诊断输入

| 维度 | 必看字段 | 判断问题 |
| --- | --- | --- |
| 时间趋势 | daily spend、install、CPI/CPA、ROI0/3/7、profit | 单日波动还是持续变化 |
| 平台状态 | serving / delivery / learning / limited / rejected / paused | 是否处于不可直接判断阶段 |
| 预算与出价 | daily budget、campaign budget、adset budget、target CPA、target ROAS、最近改动 | 波动是否来自预算或目标变更 |
| 国家 | country spend、CPI/CPA、ROI、留存、广告/IAP 收入 | 是整体问题还是国家 mix 问题 |
| 子层级 | asset group、ad group、adset、placement、device | 预算是否集中到低质子层级 |
| 素材 | asset / creative、CTR、CPC、CPM、CVR、frequency、hook、素材状态 | 是疲劳、承诺错配、审核问题还是流量变贵 |
| 用户质量 | D1/D3/D7 留存、关键事件、广告展示、付费/订阅行为 | 低成本是否带来低质用户 |
| 收入结构 | ads revenue、IAP revenue、subscription revenue、profit | IAA 与 IAP 是否互相掩盖问题 |

## 9. 生命周期判断矩阵

| 信号 | testing | scaling | mature | fatigue | pause_candidate |
| --- | --- | --- | --- | --- | --- |
| Spend / install | 未达样本门槛 | 样本充足且持续增长 | 稳定 | 增长停滞或成本抬升 | 持续消耗但无有效回收 |
| CPI / CPA | 可波动 | 达标或可接受 | 稳定 | 连续上升 | 连续不达标 |
| ROI / profit | 尚未完整 | 达标或边际可接受 | 稳定达标 | 下降 | 连续不达标 |
| 留存 / 事件 | 未完整 | 不低于阈值 | 稳定 | 下滑 | 明显低质 |
| 素材 / 资产 | 新素材学习 | winner 或有效资产支撑 | 需要储备变体 | 频次、CTR/CVR 或资产质量恶化 | 刷新无效 |
| 平台学习状态 | 可能 learning | learning 风险可控 | 稳定 | 频繁重新学习或受限 | 非学习问题导致持续差 |

## 11. 动作规则

| 动作 | 使用条件 | 复查窗口 | 注意事项 |
| --- | --- | --- | --- |
| `test_budget` | 新平台结构、新国家、新素材、新目标事件，样本不足但值得验证 | 2-3 个完整自然日或达到样本门槛 | 不把测试预算当成放量预算 |
| `maintain` | 回收可接受但边际空间不清晰，或平台仍处于学习/观察阶段 | 24 小时到下个日报 | 必须写复查日期 |
| `small_add` | 回收达标但仍有 forecast、样本、学习状态或国家结构风险 | 2-3 个完整自然日 | 优先小幅验证，观察边际回收 |
| `add_budget` | 多个完整窗口达标，数据质量 ready，素材/资产供给充足 | 下一个完整自然日开始连续复查 | 不因平台 CTR/CPI/ROAS 单项好就放量 |
| `refresh_creative` | 素材疲劳、频次上升、CTR/CVR 下降、承诺错配、资产不足 | 2-3 个完整自然日 | 明确刷新方向：hook、first frame、CTA、语言、场景、权益 |
| `reduce_or_reallocate` | 成熟对象回收持续弱，且存在更好预算承接对象 | 下一个完整自然日 | 数据 blocked 时不得直接降预算 |
| `pause_candidate` | 连续不达标，刷新无效，排除学习期、归因、收入延迟和样本不足 | 人工确认后执行 | 本页只给候选，不自动暂停 |
| `data_gap_check` | 平台与 Snowball / BI 对账失败 | 立即 | 修复前不做强投放判断 |

动作的进入条件以 [[动作分级与决策边界]] 为准。Google 平台子动作约定：`add_assets`、`refresh_video`、`split_asset_group` 属于 `refresh_creative` 的 Google 专用细分；观察类结论统一写 `maintain`。

## 14. 注意事项

- Google Ads 和 Meta Ads 的平台机制不同，总模板不能替代平台专属文档。
- Google 的 `asset performance label`、Meta 的 creative CTR / frequency 都不是 creative 级真实 ROI。
- Snowball 当前通常不到 creative 级 ROI；素材回收必须写成“基于所属 campaign / adgroup / adset / asset group 回收质量推断”。
- ROI7 若为预测值，必须标注 `forecast`。
- 当天数据、学习期数据、样本不足数据只能作为低置信度信号。
- 平台写操作，如加预算、暂停、改出价、改目标事件、创建广告，必须有明确授权和二次确认。
