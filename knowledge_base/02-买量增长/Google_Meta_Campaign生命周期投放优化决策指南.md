# Google / Facebook Campaign 生命周期投放优化决策指南

基于《Campaign 生命周期业务规则 V3 最终业务规则版》和 paid-ads 投放优化框架整理。

## 0. 定位与词表映射

本页是买量决策的量化规则层：把 [[广告系列生命周期诊断模板]] 的通用阶段框架细化为可执行的每日判断顺序、量化阈值和单主动作输出。日常执行流程和参数维护见 [[买量每日决策SOP与参数表]]；动作词表唯一定义见 [[动作分级与决策边界]]；平台机制细节见 [[Google投放与诊断手册]] 和 [[Meta投放与诊断手册]]。

### 0.1 生命周期阶段映射

本指南的五阶段与知识库标准五状态对应如下，日报和诊断输出统一用标准词，本指南阶段名可作为 `sub_stage` 补充：

| 本指南阶段 | 知识库标准状态 | 说明 |
| --- | --- | --- |
| 测试期 | testing | 未完成有效验证，样本未成熟 |
| 放量验证 | testing -> scaling 过渡（sub_stage: scaling_trial） | 首次 +10%-20% 后观察 1-2 个 conversion cycles |
| 放量期 | scaling | 已验证可承接更大预算 |
| 平稳期 | mature | 可维持或长尾经营 |
| 衰退期 | fatigue（前端疲劳 / 库存恶化 / 结构老化）或 pause_candidate（商业失效） | 按主因四分法拆到标准状态 |

### 0.2 动作词映射

| 本指南动作 | 知识库标准动作 | 说明 |
| --- | --- | --- |
| 继续观察 | maintain | 必须写复查窗口 |
| 加预算 | small_add（+10%-15%）/ add_budget（+15% 以上） | 按幅度和置信度区分 |
| 回退预算 | reduce_or_reallocate | 默认回退 10%-20% |
| 换素材 | refresh_creative | 含 Google 子动作 add_assets / refresh_video |
| 复制重启 | duplicate_restart | 复制后视为新 testing，不继承原判断 |
| 关停 | pause_candidate | 本指南只给候选，人工确认后执行 |

### 0.3 指标口径映射

本指南以首付类事件为核心指标（适合 IAP / 首充类产品）；IAA 产品套用时按下表替换：

| 本指南指标 | 含义 | IAA 产品对应口径 |
| --- | --- | --- |
| FCPA | 首付用户获取成本 | CPI 或关键事件 CPA |
| FPR | 点击/安装到首付的转化率 | 关键事件转化率、广告展示渗透（IPU） |
| ROAS1 / ROI1 | 首日回收 | ROI0（Snowball 口径） |
| 首付人数 | 量级承接指标 | install 数或关键事件人数 |
| conversion cycle | 优化事件的完整转化周期 | 通常 1-2 个完整自然日；lag-heavy 目标更长 |

## 1. 核心理解

这份规则文档的本质，不是让系统替代投放优化师自动调预算，而是把优秀优化师每天判断 campaign 的思路标准化：

1. 先确认 campaign 是否真的还在投放。
2. 再确认平台是否处于学习期、审核恢复、重大编辑重置、转化滞后保护等不可强判状态。
3. 再判断当前问题属于前端、流量库存、商业结果、还是操作后恢复窗口。
4. 最后才判断 campaign 处于测试期、放量期、平稳期或衰退期。
5. 每天每条 campaign 只给一个主动作，避免同时建议降预算、换素材、复制重启、关停，导致投手无法执行和复盘。

因此，这套规则适合做“日报第一轮判断系统”：系统每天先给生命周期、原因和动作建议，业务侧再对重点分歧拍板，随后把高频分歧沉淀为规则参数。

## 2. 决策总原则

| 原则 | 解释 | 投放执行含义 |
| --- | --- | --- |
| 保护层优先 | delivery_status、platform_status、异常事件优先于商业判断 | 学习期、恢复期、重大编辑后，不直接强关停或强放量 |
| 成熟窗口优先 | 12-24h 只能预警，强动作要等 1-2 个 conversion cycles | Google/value 型 campaign 更要避免过早下结论 |
| 单主动作 | 每天每条 campaign 只输出一个最优动作 | 便于投手执行，也便于第二天复盘动作效果 |
| 前端与后端分离 | CTR/eCPM 诊断流量与素材，FCPA/ROAS/ROI 诊断商业承接 | CTR 低不能单独作为商业关停理由 |
| 操作记录先于结果数据 | 刚降预算、换素材、新建广告组、过审恢复时先观察 | 避免把人为操作后的波动误判为自然衰退 |
| Google 看 campaign + 广告组 | 不只看 campaign 预算，要判断是否局部广告组问题 | 优先局部换素材、新建/复制广告组，再考虑整盘降预算 |
| Facebook 看 campaign + ad | Meta 更依赖素材和广告层表现 | 素材疲劳时优先换广告，老结构失效后复制重启 |

## 3. 生命周期阶段与投手动作

| 阶段 | 业务含义 | 可以做的动作 | 禁止或谨慎动作 |
| --- | --- | --- | --- |
| 测试期 | 尚未完成有效放量验证 | 硬止损、软止损/观察、进入放量验证 | 不因单一 FCPA 偏高就强关停 |
| 放量验证 | 从测试期进入正式放量前的验证动作 | +10%-20%，观察 1-2 个 conversion cycles | 不用 12-24h 早期波动定义失败 |
| 放量期 | 已验证可承接更大预算 | 高弹性 +20%-30%，中弹性 +10%-15%，低弹性回退 | 不连续两次 >20% 大幅上调 |
| 平稳期 | 已进入可维持或长尾经营状态 | 维持预算，高质量长尾可机会性 +10%-20% | 异常周、赛事周不要用常规波动阈值硬判 |
| 衰退期 | 成熟盘出现持续恶化 | 换素材、降预算、复制重启、关停四选一 | 不在保护窗内混合输出多个动作 |

## 4. Google Ads 实践决策思路

Google 的关键不是只看 campaign 层预算，而是同时看广告组、素材、出价目标和学习状态。尤其是 UAC / App 类投放，单个 campaign 里不同广告组或主题的承接差异会很大。

### 4.1 Google 每日判断顺序

1. 看最新日期是否有消耗。无消耗的 campaign 不进入当日日报。
2. 看是否 paused、disapproved、申诉恢复、近 3 天无消耗。
3. 看是否 learning、learning_limited、significant_edit_reset、conversion_lag_protected。
4. 看最近 48-72h 是否有预算调整、新建广告组、启用广告组、素材更新、出价目标调整。
5. 判断问题是个别广告组弱，还是多个广告组同时承接变差。
6. 再进入生命周期判断和动作输出。

### 4.2 Google 动作优先级

| 场景 | 典型信号 | 推荐动作 |
| --- | --- | --- |
| 刚做关键操作 | 48h 内降预算，72h 内新建/启用广告组或改素材 | 继续观察 24-72h |
| 个别广告组弱 | 弱广告组 CTR/FCPA/ROAS 明显差，主力广告组仍能承接 | 弱广告组换素材，campaign 预算先维持 |
| 当前主题跑不动但历史有强主题 | 某主题衰退，历史优胜广告组仍有参考价值 | 复制优胜广告组或新建主题广告组，小预算测试 |
| 多广告组同时变差 | 前端没有单点问题，FCPA/ROAS/首付人数整体走弱 | campaign 回退预算 10%-20% |
| 大盘老化 | 14 天内已试过广告组复制、新建、换素材、降预算仍无恢复 | 复制 campaign 重启 |
| 整盘失效 | 主力广告组和整盘都明显失效，且不在保护窗 | 关停 |

### 4.3 Google 放量判断

Google 一般比 Facebook 更需要耐心，尤其是 value / ROAS / lag-heavy 优化目标。规则里 Google 最早正式复核日是第 6 天，说明它默认更依赖成熟窗口。

放量前必须确认：

- delivery_status = active_delivery。
- platform_status = active。
- 最近 1 个 conversion cycle 没有重大编辑。
- 近 2 天首付连续。
- FCPA <= 1.25 x 目标 FCPA。
- ROAS1 或 ROI1 >= 0.85 x 目标。
- CTR、eCPM、FPR 至少 2 项接近基线。

放量动作建议：

- 普通候选：+10%-15%。
- 高置信候选：+15%-20%。
- 已进入高弹性放量：+20%-30%。
- 每 1 个 conversion cycle 最多一次放量。

## 5. Facebook / Meta 实践决策思路

Facebook 的核心是 campaign + ad 语境。Meta 更依赖素材、学习状态、广告审核、受众承接和预算放大后的流量质量变化。

### 5.1 Facebook 每日判断顺序

1. 看最新日期是否有消耗，无消耗不进入当日日报。
2. 看 campaign/ad/ad set 是否刚过审、刚恢复、刚暂停或刚调预算。
3. 看最近 72h 是否有 Ad created / Ad updated / ad set 修改 / bid strategy 修改。
4. 如果刚做动作，优先观察，不重复建议同类动作。
5. 判断是素材疲劳，还是预算放大后商业承接变差。
6. 判断是否进入老结构失效，需要复制重启或关闭。

### 5.2 Facebook 动作优先级

| 场景 | 典型信号 | 推荐动作 |
| --- | --- | --- |
| 刚做关键动作 | 48h 内降预算，72h 内换广告、新建广告、改 ad set 或 bid | 继续观察 24-72h |
| 素材疲劳 | campaign 仍有消耗，CTR 或点击后表现变差，72h 内没有新广告 | 换素材 |
| 放大后承接变差 | 前端没明显坏，FCPA/ROI/ROAS/首付人数走弱 | 回退预算 10%-20% |
| 老结构老化 | 14 天内已降预算、换素材/新广告，仍未恢复 | 复制重启 |
| 恢复概率低 | 商业结果明显失效，且不在审核/操作保护窗 | 关停 |

### 5.3 Facebook 测试期额外关注

Meta 进入放量验证前，需要额外确认：

- live creative 数量 >= 2。
- top creative spend share <= 75%。

原因是 Meta 过度依赖单一素材时，短期 ROI 可能好看，但放量后容易出现素材疲劳、CPM 上升、CTR 下降和首付人数增长不匹配。

## 6. 关键指标解释

| 指标 | 用途 | 错误用法 |
| --- | --- | --- |
| CTR | 判断前端素材/流量兴趣 | 单独用于商业关停 |
| eCPM | 判断竞争、库存和平台流量成本 | 不结合 CTR 与转化直接判断坏盘 |
| FPR | 判断点击到首付转化承接 | 不看样本量就判断趋势 |
| FCPA | 判断获客成本 | 测试期单日偏高就关停 |
| ROAS1 / ROI1 | 判断首日回收质量 | conversion lag 未完成就强判 |
| 首付人数 | 判断量级承接 | 不结合预算增长率判断放量成功 |

## 7. 典型决策模板

### 7.1 测试期硬止损

满足以下任意一类才建议强关：

- 样本已成熟，CTR < 0.6 x CTR_base，且 eCPM > 1.3 x eCPM_base。
- mature_window 完成，累计消耗 >= 1.2 x T_FCPA，首付人数 = 0，且不在转化滞后保护。
- mature_window 完成后，FCPA、FPR、ROI1、ROAS1、连续首付趋势至少 3 项共振失败。

执行理解：测试期不能因为 FCPA 单点难看就关。要么前端严重失败，要么成熟后完全没转化，要么多信号同时坏。

### 7.2 进入放量验证

必须同时具备：

- 投放正常，平台 active。
- 近 1 个 conversion cycle 没有重大编辑。
- 近 2 天首付连续。
- FCPA <= 1.25 x 目标。
- ROAS1 或 ROI1 >= 0.85 x 目标。
- CTR/eCPM/FPR 至少两项支持。

动作：先 +10%-15%，高置信才 +15%-20%。

### 7.3 放量成功

放量后等待 1-2 个 conversion cycles，满足：

- CTR 下降 <= 15%。
- FCPA 上涨 <= 25%。
- ROAS1 或 ROI1 保留到放量前 80% 以上。
- 首付人数增长 >= 预算增长 x 40%。

动作：进入正式放量期。

### 7.4 放量失败

满足以下任意 2 条：

- CTR 下降 >= 25%。
- FCPA 上涨 >= 60%。
- ROAS1 或 ROI1 下降 >= 40%。
- 首付人数增长 < 预算增长 x 25%。
- eCPM 上涨 > 35%。

动作：回退预算 10%-20%，然后进入回退后观察。

### 7.5 衰退期主因判断

| 主因 | 判断重点 | 主动作 |
| --- | --- | --- |
| 前端疲劳 | CTR 较高点下降 >30%，但 FPR 下降 <=20% | 换素材 |
| 库存/流量恶化 | eCPM 上升，首付承接变差，多点走弱 | 降预算观察 |
| 结构老化 | 历史跑通过，降预算和换素材后仍恢复有限 | 复制重启 |
| 商业失效 | CTR、FCPA、ROAS、首付人数多项极端恶化 | 立即关闭 |

## 8. 日报输出建议

每天每条 campaign 至少输出：

| 字段 | 业务解释 |
| --- | --- |
| delivery_status | 是否真实在投放 |
| platform_status | 是否处于学习、限制、重大编辑、转化滞后保护 |
| context_tag | 重大事件、支付异常、风控、产品事故、素材疲劳等 |
| problem_layer | 当前主要问题层级 |
| business_stage | 测试期、放量期、平稳期、衰退期 |
| sub_stage | 高弹性放量、放量失败平稳、前端疲劳等子类 |
| action_type | 继续观察、加预算、回退预算、换素材、复制重启、关停 |
| reason_summary | 一句话解释为什么 |
| trigger_reason | 命中的规则条件 |
| observe_window | 下次观察窗口 |
| confidence_level | 高、中、低置信 |

日报摘要建议按以下顺序给业务：

1. 今日应加预算 campaign。
2. 今日应降预算或止损 campaign。
3. 今日需要换素材 / 新建广告组 campaign。
4. 今日继续观察但风险较高 campaign。
5. 不进入判断的无消耗 / 已停投 campaign 数量。

## 9. 对公司当前投放管理的落地建议

### 9.1 第一阶段：先做判断系统，不做自动执行

文档明确不包含自动调预算、自动关停、自动建 campaign。第一版应只做：

- 拉取主效果表。
- 拉取 Google 更改历史。
- 拉取 Facebook Activity history。
- 拉取重大事件表。
- 标准化为 campaign_ops_log 和 major_event_log。
- 输出生命周期、原因和主动作。
- 回写 Lark Sheets 并推送机器人日报。

### 9.2 第二阶段：沉淀投手反馈

每天业务侧只需要反馈两类：

- 认同。
- 不认同，并说明实际动作和原因。

系统要重点沉淀这些分歧：

- 哪些 Google campaign 因转化滞后被系统误杀。
- 哪些 Meta campaign 因素材集中度过高导致放量失败。
- 哪些重大赛事或本地事件应放宽阈值。
- 哪些分包渠道 x 媒体需要更宽或更严的阈值。

### 9.3 第三阶段：参数化不同媒体

默认阈值只能作为起点。建议按分包渠道 x 媒体维护参数：

- T_FCPA。
- T_ROAS1。
- T_ROI1。
- CTR_base。
- eCPM_base。
- FPR_base。
- conversion_cycle。
- 放量幅度上限。
- 异常周阈值放宽比例。

Google / value / lag-heavy 类型应比 Meta / firstcharge 更宽容。

## 10. 优化师最终执行口诀

1. 没消耗，不判断。
2. 学习期、恢复期、重大编辑后，先保护。
3. 12-24h 只预警，不强判。
4. 测试期看成熟样本，不因单点 FCPA 关停。
5. 放量看承接，不只看 ROAS。
6. Google 先看广告组局部问题，再动 campaign。
7. Facebook 先看素材和审核恢复，再动预算。
8. 衰退期先找主因，每天只给一个主动作。
9. 强关停必须排除异常保护窗。
10. 系统先判断，投手拍板分歧，规则持续沉淀。
