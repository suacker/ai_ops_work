# 预算迁移决策模板

## 使用场景

当多个国家、campaign、adgroup/adset 或渠道之间出现回收差异时，用本模板判断预算应加到哪里、从哪里降、是否只做小幅验证，或是否需要先修复数据。预算迁移是建议流程，默认不直接执行平台写操作。

## 候选对象

| 方向 | 对象 | 原因 | 必要条件 | 风险 | 建议动作 |
| --- | --- | --- | --- | --- | --- |
| 加预算 |  | 回收稳定、利润正、质量达标 | 数据质量 ready，观察窗完整 | 放量后边际回收下降 | add_budget / small_add |
| 保持 |  | 回收可接受但增量不清晰 | 需要更多样本或素材储备 | 错过增量机会 | maintain |
| 降预算 |  | 回收弱或机会成本高 | 非归因问题，连续不达标 | 误杀恢复中对象 | reduce_or_reallocate |

| 数据修复 |  | 映射或回收缺失 | 先修复数据 | 延迟决策 | data_gap_check |

## 迁移规则

| 规则 | 说明 |
| --- | --- |
| 先同层迁移 | 优先在同渠道、同国家、同目标内迁移，减少结构变量 |
| 再跨国家迁移 | 国家阈值和变现差异明确时，再从弱国家迁到强国家 |
| 最后跨渠道迁移 | Google 和 Meta 用户结构不同，跨渠道迁移要重新观察质量 |
| 小步快跑 | 不稳定对象优先 `small_add`，不要一次性大幅上调 |
| 留出复查 | 每次迁移必须写复查日期和失败回滚条件 |
