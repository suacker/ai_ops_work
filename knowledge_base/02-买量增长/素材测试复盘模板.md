# 素材测试复盘模板

## 使用场景

用于复盘 Meta creative、Google asset、视频 hook、首帧、卖点、CTA、字幕、本地化版本等素材测试。素材判断必须结合所属 campaign/adgroup 的 Snowball / BI 回收质量，不得只按 CTR、CPC 或平台转化率判 winner。

## 判断规则

| 判断 | 条件 | 后续动作 |
| --- | --- | --- |
| winner | 样本足够，CTR/CVR/CPI 表现好，且 parent 回收和用户质量达标 | 扩展 hook、首帧、CTA、本地化变体 |
| scalable | 已验证 winner，多个观察窗回收稳定，预算增加后未明显劣化 | 进入 `small_add` 或 `add_budget` 候选 |
| keep_testing | 指标有亮点但样本不足或回收未完整 | 保持测试预算，补观察窗 |
| refresh_needed | 频次升高、CTR/CVR 下滑、CPI 上升但回收仍有基础 | 做新变体，不急停 parent |
| pause_candidate | 样本足够但平台效率和 parent 回收持续不达标 | 人工确认后停止复用 |
| mismatch | CTR 高但 ROI、留存或关键事件弱 | 重写承诺、落地页或目标人群 |

## 注意事项

- Snowball 当前通常不到 creative 级 ROI，creative 回收必须写成基于所属 Campaign/Adgroup 推断。
- 高 CTR 低 ROI 可能是素材承诺错配或低质量流量，不应直接放量。
- Google App Campaign / PMax 的 asset 表现容易受系统组合影响，不要把单 asset 的平台标签直接等同于真实 ROI。
- fresh low-spend 新素材必须标记 `too early to judge`。
