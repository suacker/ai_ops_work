# 产品资产治理规则

## 治理目标

产品资产治理的目标是让任何经营分析和执行动作都能快速找到正确对象：Project ID、账号、包名、商店页、广告单元、数据看板、权限 Owner 和历史变更。

## 资产分类

| 分类 | 具体资产 | 主 Owner | 更新触发 |
| --- | --- | --- | --- |
| 产品主数据 | Project ID、产品名、平台、包名、变现类型、阶段 | 产品负责人 | 新产品、改名、包名变化、阶段变化 |
| 买量资产 | 账号、Campaign 命名、素材库、落地页/商店页链接 | UA 负责人 | 新账号、新渠道、新素材批次 |
| 变现资产 | AdMob/AppLovin 应用、广告单元、场景、mediation 配置 | 变现负责人 | 新广告位、配置调整、收入异常 |
| 订阅资产 | Paywall、价格、权益、订阅组、退款规则 | 产品负责人 | 价格/权益/Paywall 改版 |
| 数据资产 | Snowball 项目、事件、漏斗、看板、指标口径 | BI 负责人 | 埋点、报表、口径变化 |
| 权限资产 | 后台账号、权限级别、审批记录、交接记录 | 工具/权限 Owner | 入职、离职、转岗、临时授权 |

## 资产生命周期

| 阶段 | 必须完成 | 退出条件 |
| --- | --- | --- |
| 创建 | 建立产品索引、账号索引、数据入口、Owner | 可被经营会议引用 |
| 使用 | 有状态、负责人、最近更新时间 | 可支持日常分析和动作追踪 |
| 变更 | 记录变更原因、影响范围、回滚方式 | 相关报告和权限同步完成 |
| 废弃 | 标记 archived，保留历史链接 | 无活跃预算、无活跃实验、权限已回收 |

## Owner 机制

| 规则 | 说明 |
| --- | --- |
| 单一最终 Owner | 每类资产只能有一个最终负责人，避免多人都以为别人维护 |
| 协作方可多选 | UA、变现、产品、BI 可以作为协作方 |
| 变更先更新索引 | 先更新资产页，再在专题报告中引用 |
| 月度抽检 | 每月抽检高收入产品、高花费账号和高风险权限 |

| 检查项 | 通过标准 | 结果 | 修复 Owner | 截止日期 |
| --- | --- | --- | --- | --- |

| 账号权限准确 | 权限 Owner、可操作人、到期日齐全 |  |  | YYYY-MM-DD |
| 商店页资产可访问 | 云盘链接和 live 版本一致 |  |  | YYYY-MM-DD |
