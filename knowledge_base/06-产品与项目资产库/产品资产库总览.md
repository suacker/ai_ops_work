# 06 产品与项目资产库

## 定位

产品与项目资产库是全员入口，解决“产品是什么、账号在哪里、商店页资产在哪里、谁有权限、谁负责维护”的问题。它是经营分析、买量执行、变现实验、版本发布和人员交接的基础设施。

## 资产治理原则

- 文档只保存索引、Owner、账号 ID、链接和状态，不保存 token、密码、secret。
- Project ID、包名、Bundle ID、账号 ID、商店链接、数据看板链接必须可追溯。
- 一个资产只能有一个最终 Owner，可以有多个协作方。
- 资产变更必须写日期、变更人、变更原因和影响范围。
- 任何离职、转岗、外包交接必须先完成权限回收和 Owner 更新。

## 资产完整度检查

| 检查项 | 必填字段 | 缺失风险 |
| --- | --- | --- |
| 产品基础信息 | Project ID、产品名、平台、包名/Bundle ID、变现类型、阶段、Owner | 分析口径混乱 |
| 买量账号 | 平台、Account ID、账号名、权限 Owner、可操作人 | 预算动作无法追责 |
| 变现账号 | AdMob/AppLovin 应用、广告单元、mediation 入口、Owner | 收入异常无法定位 |
| 商店资产 | 国家/语言、标题、描述、截图、视频、关键词、链接 | ASO 和素材复用困难 |
| 数据入口 | Snowball/BI 项目、核心看板、事件表、漏斗链接 | 报告无法复现 |
| 权限记录 | 申请人、审批人、权限级别、到期日 | 安全和交接风险 |
