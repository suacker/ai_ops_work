# IAP 与 iOS 订阅经营看板模板

## 适用场景

用于订阅型或 IAP 产品的周报、月报、异常排查和买量质量复盘。iOS 订阅必须单独分析，不要和 Android IAA 汇总成一个收入结论。

本模板不填写实时数据。执行时从 App Store Connect、Snowball、BI、支付平台或归因平台拉取真实数据。

漏斗判断规则：

- Paywall 曝光低：优先查入口触发、用户分层、冷启动/核心流程覆盖。
- 点击购买低：优先查权益表达、价格锚点、CTA、视觉层级、信任信息。
- 支付拉起低：优先查 StoreKit、商品配置、地区价格、客户端错误。
- Trial start 高但转正低：优先查试用承诺、权益留存、提醒、用户质量。
- 首购高但续订低：说明短期销售强但长期价值弱，不能直接扩大买量。
- 退款高：优先查素材承诺、Paywall 文案、误触、价格展示、取消路径和客服反馈。

cohort 分析要求：

- 订阅收入必须按安装 cohort 看，不只按交易日期看。
- iOS 买量判断至少同时看 `first_purchase_rate_by_install`、`first_purchase_cost_by_install`、`D0/D7/D30 LTV` 和预测 ROI。
- 试用期较长时，D0 收入不能代表最终质量，需要标注未成熟 cohort。
- 续订收入回收慢的产品，需拆 mature cohort 与 immature cohort。

| Yearly | 1 年 | US |  | Yes / No |  |  |  |  |  |  |  |

价格判断规则：

- Weekly 转化高但退款高：可能是价格感知、误触或低质量流量问题。
- Yearly 收入高但转化低：检查年包折扣表达、年省金额、默认选中策略。
- Trial 商品转正低：优先优化试用期产品价值触达，不要只缩短试用。
- 国家价格异常：检查本地购买力、竞品价格、税费、汇率和 App Store 价格层级。

国家结论要求：

- 高收入国家优先拆渠道和 campaign，不要只看国家均值。
- 退款率、取消率和续订率要结合国家文化、支付习惯和价格层级解释。
- IAP 与订阅并存时，要区分一次性购买、订阅首购和续订，不要合并为 `purchase`。
