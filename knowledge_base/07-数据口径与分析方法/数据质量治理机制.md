# 数据质量治理

## 治理目标

数据质量治理用于确保经营会议、预算动作、变现实验和产品决策基于可复现数据。任何会影响动作判断的数据问题都必须登记、分级、指定 Owner 并复查。

## 数据质量等级

A/B/C/D 是 BI 治理侧的登记分级；报告和动作门禁一律使用 [[动作分级与决策边界]] 的 `ready / partial / blocked / inactive_no_data`，映射关系如下表固定。

| 等级 | 定义 | 示例 | 对应报告状态 | 动作边界 |
| --- | --- | --- | --- | --- |
| A | 数据完整、口径确认、样本足够、可复现 | Snowball ROI、平台 spend、广告收入均可对账 | `ready` | 可支持 P0/P1/P2 |
| B | 核心数据可用，但有预测、延迟或轻微差异 | ROI7 为 forecast，AdMob 收入为预估 | `partial`（中等置信度） | 可支持中等置信度动作 |
| C | 样本不足、字段缺失或部分归因异常 | 新 Campaign 只有 1 天数据，事件漏斗样本小 | `partial`（低置信度） | 只做观察或小范围验证 |
| D | 数据冲突严重或无法复现 | 平台 spend 与 BI 差异大且原因未知 | `blocked` | 只做数据修复 |

## 排查清单

| 场景 | 优先检查 |
| --- | --- |
| 花费差异 | 时区、账户、币种、汇率、campaign 命名、退款/抵扣 |
| install 差异 | 归因窗口、SKAN、渠道映射、首开事件、回传延迟 |
| 收入差异 | 预估/最终收入、时区、广告源、订阅退款、税费 |
| 漏斗异常 | 事件名、版本、采样、触发条件、重复上报 |
| ROI 异常 | cohort 日期、收入窗口、forecast、spend 归属 |

## 关闭标准
