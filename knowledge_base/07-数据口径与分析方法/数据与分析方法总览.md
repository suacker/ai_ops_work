# 07 数据口径与分析方法

## 定位

数据口径与分析方法空间用于统一指标定义、数据源优先级、分析 SOP、动作分级、数据质量和报告标准。它是所有经营结论的可信度底座。

## 核心原则

- Snowball/BI 是增长、回收、漏斗、留存和广告变现判断的主数据源。
- 平台后台用于补充投放侧状态，但不能单独决定强预算动作。
- `null`、`N/A`、`-1`、预测值、采样当前日必须明确处理，不能当作真实 0。
- 报告必须写绝对日期范围、数据源、查询口径、样本边界和置信度。
- 指标口径变化必须有版本记录，避免同名指标在不同报告中含义不同。

## 数据质量的两套表达

全库共用一套数据质量语义，两种表达方式各有分工，映射关系固定：

| 场景 | 使用表达 | 定义位置 |
| --- | --- | --- |
| 报告与动作门禁（日常使用） | `ready / partial / blocked / inactive_no_data` | [[动作分级与决策边界]] |
| 数据问题登记与治理（BI 内部） | A / B / C / D 治理等级 | [[数据质量治理机制]] |

映射：A → `ready`；B → `partial`（中等置信度）；C → `partial`（低置信度）；D → `blocked`。`inactive_no_data`（无有效数据）不参与治理分级，直接输出 `no_action_no_data`。报告中一律使用 `ready / partial / blocked / inactive_no_data`，不要把 A/B/C/D 写进业务结论。
