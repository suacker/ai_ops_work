# In-app 触达运营模板

## 1. 触达定位

In-app 触达用于用户已在 App 内时的即时引导，包括弹窗、底部浮层、横幅、任务卡、空状态提示、paywall 前置解释和功能教育。它比 Push 更适合承接具体场景，但也更容易打断核心路径。

## 3. 场景选择规则

| 场景 | 适合触达 | 不适合触达 |
| --- | --- | --- |
| 首次价值路径 | 解释下一步、降低空状态困惑 | 强营销、长文案 |
| 核心任务中断 | 保存进度、继续任务、错误修复 | 跳出到无关活动 |
| Paywall 前 | 解释权益和差异 | 夸大收益、隐藏价格 |
| 广告前后 | 解释奖励或减少困惑 | 连续弹窗叠加广告 |
| 订阅到期/取消 | 说明权益损失和恢复路径 | 诱导或阻断取消 |
| 活动入口 | 明确活动权益和截止时间 | 无关用户反复展示 |

## 6. 文案与交互规范

| 类型 | 规范 |
| --- | --- |
| 标题 | 只表达一个价值点，避免夸张承诺 |
| 正文 | 说明用户现在为什么需要做这个动作 |
| CTA | 使用具体动作，例如 Start、Continue、Try、Restore |
| 关闭 | 非强制场景必须可关闭，关闭后进入频控 |
| 语言 | 使用目标语言，不直译中文 |
| 无障碍 | 按钮文案明确，避免只靠颜色表达状态 |

## 8. 决策规则

| 信号 | 解释 | 动作 |
| --- | --- | --- |
| 点击率高且核心行为提升 | 场景和承诺匹配 | 保留或扩大 |
| 点击率高但后续行为低 | CTA 或落地页不匹配 | 改落地页和任务路径 |
| 关闭率高 | 打扰、时机错误或价值弱 | 换时机、改文案、降频 |
| 投诉/卸载上升 | 触达伤害体验 | 立即停发并复盘 |
| 收入提升但退款上升 | 促销或订阅表达风险 | 降强度，检查合规 |
| 高价值用户负反馈 | 触达策略未保护价值用户 | 排除或改为低打扰样式 |
