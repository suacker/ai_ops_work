# 广告体验投诉处理 SOP

## 1. 适用范围

本 SOP 用于处理 IAA 和混合变现产品中的广告体验投诉，包括广告过多、无法关闭、低俗/违规广告、激励广告未发奖、广告导致崩溃、广告遮挡核心功能、付费用户仍看到广告等问题。

广告收入分析必须和用户体验分开判断。不能因为广告收入短期上升而忽略投诉、差评、卸载和留存损伤。

## 2. 投诉分级

| 优先级 | 场景 | 默认动作 |
| --- | --- | --- |
| P0 | 广告无法关闭、违规/低俗广告、激励广告不给奖励、大面积崩溃、付费用户仍展示广告 | 当日排查，必要时临时降级或屏蔽广告源/场景 |
| P1 | 广告过多、广告遮挡、误触高、某国家/版本集中投诉 | 1-3 个完整自然日内定位并调整频控或场景 |
| P2 | 单点体验建议、低频素材不适、轻微文案误解 | 纳入优化池 |

## 3. 信息收集表

| 字段 | 说明 |
| --- | --- |
| user_id / device_id | 用户定位 |
| platform / app_version | 平台和版本 |
| country / language | 国家和语言 |
| complaint_time | 投诉时间，写绝对时间和时区 |
| ad_format | AppOpen / Interstitial / RewardedVideo / Banner / Native |
| placement / scene | 广告场景 |
| ad_source / mediation_source | AdMob / AppLovin / network |
| ad_unit_id | 广告单元 |
| creative / advertiser | 如可获得 |
| screenshot / screen_recording | 用户证据 |
| session_id | 排查广告链路 |
| reward_event | 激励广告奖励事件 |
| subscription_status | 是否付费免广告 |

## 4. 处理流程

| 步骤 | 动作 | 负责人 | 输出 |
| --- | --- | --- | --- |
| 1. 分类 | 判断 P0/P1/P2，记录投诉类型 | 客服/运营 | 投诉卡片 |
| 2. 补证据 | 索取截图、录屏、时间、设备、国家 | 客服 | 可复现信息 |
| 3. 定位 | 查 ad_format、placement、ad_source、版本、国家 | 运营/数据 | 初步定位 |
| 4. 保护 | P0 时先降频、停场景或屏蔽高风险源 | 运营/变现 | 临时方案 |
| 5. 修复 | 调整频控、广告源、场景、回调或版本缺陷 | 变现/研发 | 修复记录 |
| 6. 回复 | 告知用户处理进展和必要补偿 | 客服 | 回复记录 |
| 7. 复查 | 看投诉、差评、广告收入、留存和激励发奖 | 运营/数据 | 复盘结论 |

## 5. 排查矩阵

| 问题 | 首查维度 | 可能原因 | 动作 |
| --- | --- | --- | --- |
| 广告无法关闭 | ad_format、ad_source、creative、国家 | 素材关闭按钮违规、SDK 问题 | 屏蔽素材/广告源，反馈平台 |
| 广告过多 | placement、IPU、show_count、session | 频控过高、场景叠加 | 降频、互斥、保护新用户 |
| 低俗/违规广告 | ad_source、creative、国家、截图 | 广告审核漏出 | 屏蔽 advertiser/creative，提交平台 |
| 激励广告不给奖励 | reward_callback、server_verify、network | 回调丢失、网络失败、重复领取 | 补偿用户，修复回调 |
| 广告导致崩溃 | app_version、SDK、ad_format、crash log | SDK/适配器问题 | 降级广告源或版本修复 |
| 付费用户仍有广告 | subscription_status、entitlement、cache | 权益同步失败 | 修复权益同步，补偿 |
| 广告遮挡功能 | placement、屏幕尺寸、语言长度 | UI 适配问题 | 改布局或禁用场景 |

## 6. 体验护栏

| 用户层 | 广告策略 |
| --- | --- |
| 新用户首日 | 控制 AppOpen/Interstitial 频次，避免打断首个核心行为 |
| 激活未稳定用户 | 广告放在自然结束点，不打断任务中段 |
| 高价值/订阅用户 | 应减少广告或完全免广告，按权益定义执行 |
| 投诉用户 | 进入冷却或排除，避免重复刺激 |
| 低留存 cohort | 先排查广告干扰，再判断是否流量质量问题 |

## 8. 决策规则

| 信号 | 动作 |
| --- | --- |
| P0 投诉可复现 | 先保护用户体验，再复查收入影响 |
| 投诉集中于单一广告源 | 临时屏蔽或降权该源，反馈平台 |
| 投诉集中于单一场景 | 降频或调整触发时机 |
| 收入下降但投诉显著下降、留存改善 | 保留体验优化，评估长期 LTV |
| 收入上升但投诉/卸载/差评上升 | 不直接放大，先做护栏修复 |
| 缺少广告源或场景字段 | 标记数据缺口，优先补日志 |

## 9. 用户回复要点

| 场景 | 回复要点 |
| --- | --- |
| 广告无法关闭 | 致歉，索取截图/录屏/时间/设备，说明会定位广告来源 |
| 违规广告 | 致歉，要求截图或广告内容，说明会屏蔽并反馈平台 |
| 激励未到账 | 确认账号、时间和奖励内容，符合条件时补偿 |
| 付费仍有广告 | 确认订单和权益，优先恢复权益并补偿 |
