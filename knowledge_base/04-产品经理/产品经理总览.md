# 03 产品经理

## 定位

产品经理空间用于沉淀出海 App 的产品定位、Roadmap、PRD、实验设计、版本复盘、指标体系、竞品研究和商业化决策。

这里的 PM 不是只写需求的人，而是负责把用户价值、留存、变现、买量回收和版本质量连接成可执行决策的人。所有文档都应能支持团队回答：

- 这个产品服务谁，解决什么高频或高价值问题？
- 当前阶段最该提升的是留存、付费、广告收入、买量回收，还是版本质量？
- 需求如何影响指标树，是否值得投入研发、设计、投放和运营资源？
- 上线后如何判断成功、失败、继续观察或回滚？

## PM 关注指标

| 领域 | 核心指标 | PM 判断问题 |
| --- | --- | --- |
| 新用户 | 首次打开、onboarding 完成、核心功能首次使用、首日关键行为次数 | 用户是否在 1 天内理解产品价值？ |
| 活跃 | DAU、WAU、使用频次、核心行为次数、功能渗透率 | 用户是否形成可重复使用场景？ |
| 留存 | D1、D3、D7、D30、回访间隔、流失节点 | 产品价值是否足以支撑买量和订阅？ |
| 订阅/IAP | Paywall CVR、Trial Start、Trial to Paid、续订率、退款率、LTV | 付费承诺是否与实际体验匹配？ |
| IAA | Ads ARPDAU、impressions/user、IPU、eCPM、广告触发率、跳出率 | 广告库存是否增加收入且不破坏体验？ |
| 买量回收 | CPI、CPA、ROI0、ROI3、ROI7、国家/渠道/cohort profit | 产品承接是否支撑继续买量？ |
| 质量 | Crash Free、ANR、支付失败、订阅恢复失败、客服反馈 | 版本质量是否影响转化、留存和收入？ |

## 决策原则

- 需求优先级必须写清业务目标、目标用户、影响指标、机会成本和不做的后果。
- 实验不是上线记录，必须包含假设、成功标准、观察窗口、护栏指标和复盘决策。
- iOS 订阅型产品的产品决策要同时关注转化率、试用转正、续订率、退款率和客服投诉。
- IAA 产品不能只看短期 Ads ARPDAU，必须同时检查留存、使用深度、广告退出率和买量回收。
- 混合变现产品要明确用户分层：广告用户、试用用户、付费用户、高 LTV 用户不能用同一套体验策略。
- 所有指标结论必须标注数据源、口径和日期窗口；没有实时数据时只能写方法、模板和示例占位。

## 标准工作流

```text
产品战略 -> 指标树 -> 机会识别 -> PRD/实验设计 -> 埋点验收 -> 上线监控 -> 版本复盘 -> Roadmap 调整
```

| 阶段 | PM 产物 | 关键检查 |
| --- | --- | --- |
| 战略 | 产品定位、用户分层、季度目标 | 是否聚焦一个主要增长约束 |
| 指标 | 指标树、看板字段、数据口径 | 是否能定位到国家/渠道/版本/人群 |
| 需求 | PRD、优先级评分、埋点需求 | 是否有成功标准和不做范围 |
| 实验 | 实验方案、样本和护栏 | 是否能独立判断胜负 |
| 上线 | 版本监控清单、回滚条件 | 是否覆盖质量、收入、留存、买量 |
| 复盘 | 版本复盘、后续动作 | 是否形成 P0/P1/P2 决策 |

## 常用动作分类

| 动作 | 适用场景 | PM 输出 |
| --- | --- | --- |
| 放大 | 指标达标且护栏稳定 | 扩大实验流量、进入 Roadmap、同步买量侧 |
| 继续观察 | 样本不足或短期波动 | 明确观察窗口、补数口径、下一次判断日期 |
| 调整方案 | 主指标未达预期但护栏安全 | 迭代假设、缩小人群、改交互或定价 |
| 回滚/降级 | 崩溃、退款、留存或收入护栏恶化 | 触发回滚条件、记录影响范围和修复 Owner |
| 暂停投入 | 没有用户价值或商业闭环 | 关闭需求、沉淀原因、释放资源 |
