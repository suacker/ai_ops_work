# 每日经营扫读模板

## 使用场景

用于每个工作日的日经营扫读（流程见 [[经营会议机制]]），把驾驶舱"首页第一屏"要求的 5 个区块落成当日快照。目标是 15 分钟内发现 P0 异常、确认当天动作，不做深度分析；需要下钻时链接到对应模块页面。

## 基本信息

| 项目 | 内容 |
| --- | --- |
| 日期 | YYYY-MM-DD（数据为前一完整自然日） |
| 主持人 | 增长负责人 |
| 数据源 | Snowball / BI（平台后台数据仅作上下文） |
| 数据质量 | ready / partial / blocked |

## 1. 五行经营结论

> 每行必须有动作含义，不接受纯描述。数据不足的行明确写"数据缺失"或"低置信度"。

```text
1.（收入）
2.（花费）
3.（利润 / ROI）
4.（最大风险）
5.（今日最优先动作）
```

## 扫读边界

- 只看前一完整自然日数据，当天实时波动不进入结论，除非明确标低置信度。
- 不能只凭 CTR、CPI 或平台 ROAS 给出放量结论，预算动作必须有 Snowball/BI 回收证据（见 [[动作分级与决策边界]]）。
- 广告收入异常不能只用 eCPM 解释，必须同时看 DAU、展示、IPU、广告格式和 mediation source。
- 扫读中发现的重大问题走 [[经营会议机制]] 的升级机制，不在扫读会上展开分析。
