# 三、YouDrama iOS(A148，订阅/IAP 短剧)——小盘，近期激进放量遇滞后墙，应回退到已验证水平

## 数据质量 / 保护窗 ⚠️ 这是本次最需要克制的一个
- **数据 `partial`**：iOS 仅 735/4578 安装被归因(**16%，SKAN 严重欠归因**)，cohort_revenue 被系统性低估。
- **保护窗全中**：订阅**转化滞后**(试用→转付→续订，roi7 无意义)+ **近期放量操作窗**(7-03→7-07 花费 615→721→899→1118→1275，近乎翻倍)。
- 按 KB：**iOS + lag-heavy + 刚放量，禁止用 roi7 或同日 RTS 强判失败**。

## 为什么 RTS 从 1.44 掉到 0.17 不能当放量失败铁证
订阅产品当日 RTS = 当日总收入/当日花费。你把**新增花费翻倍**，但新 cohort 的试用要几天后才转付续订，当日收入跟不上 → RTS **机械性**下滑。这正是 KB 反复强调的"放量看承接、不用同日 ROAS 判 lag-heavy"。

## 但也不能继续加 —— 单主动作 = 回退到已验证盈利水平
- 6-30~7-02 该盘在 ~$470–480/天时 RTS 稳定 1.37–1.44(盈利)；7-03 起加到 $1275/天后每天 RTS<1 且亏损扩大(7-07 亏 -1053)。
- 主力 US-2.5-260429 cohort roi0 长期仅 0.04–0.09，即便叠加 SKAN 低估和订阅滞后，CPI $1.2–1.6、CPA $33–147 的回收压力真实存在。
- JP/T1 的几条新测试(JP-2.5、T1-2.5-260702/260603)CPI $1.6–5.9、近乎 0 回收，是失败测试。

| 对象 | 单主动作 |
| --- | --- |
| US-2.5-260429(主力) | **reduce_or_reallocate：预算回退到 ~$480/天(7-01/02 已验证盈利水平)**，持一个完整转化周期(建议 7–14 天)让订阅成熟，再判 |
| JP-2.5-260702 / T1-2.5-260702 / T1-2.5-260603(新测试) | **pause_candidate**(CPI 极高、0 回收、样本已够) |
| —— 判定前必做 | 拉 **ROI14/ROI30 订阅 cohort + SKAN 校正口径**，当前 partial 数据不足以下强关停 |

**YouDrama iOS 结论**：不是"放量成功"也不是"确认失败"，是**在滞后+SKAN 迷雾里过度放量**。正确动作是把激进放量**回退到已验证的 $480/天**并等一个订阅周期成熟，同时补 ROI14/30 视图；**不要继续加**，也**不要**基于同日数据一刀切关停主力。
